Documentation for Risk Assessment Application

Business Assurance and Certification (BAC Team)

1. Overview

This application implements a risk assessment process based on ISO 31000 standards. It guides users through the steps of risk identification, analysis, and evaluation.

The application is implemented as a front-end-only solution, ensuring broad accessibility without requiring backend infrastructure.

2. File Organization

  • index.html: Main entry point to the application. Provides an introduction and navigation to the wizard.
  • wizard.html: Contains the interactive risk assessment wizard.
  • styles.css: Manages the styling of the application.
  • script.js: Handles dynamic interactivity and logic for the wizard.
  • README.txt: Instructions for setting up the application locally.

3. Technology Stack

  • Frontend: HTML, CSS, JavaScript
  • Libraries: Google Fonts, FontAwesome
  • Browser Compatibility: Modern browsers supporting HTML5 and CSS3

4. Features

  • Step-by-step wizard for conducting risk assessments.
  • Dynamic help text toggles to guide users.
  • Export of risk assessment results to a file.
  • Interactive UI with input validation.

5. Workflow

The application workflow follows these steps:

  1. Step 1: Risk Identification - Collect risk name and description.
  2. Step 2: Risk Analysis Phase 1 - Analyze impact, severity, and likelihood.
  3. Step 3: Risk Analysis Phase 2 - Assess specific risks (strategic, legal, operational, etc.).
  4. Step 4: Risk Evaluation - Summarize and export the assessment.

6. JavaScript Functions

The following JavaScript functions are implemented in the script.js file:

  • getElementById: This standard JavaScript DOM method is used throughout the script.js file to access elements by their unique ID. It is frequently used for operations such as:
    • Showing or hiding help text elements (e.g., document.getElementById("help-text").style.display).
    • Capturing user input from form fields (e.g., document.getElementById("riskName").value).
    • Updating the UI dynamically (e.g., populating summary tables).
  • toggleContent(id): Toggles the visibility of help text elements based on their ID.
  • navigateToStep(stepIndex): Navigates between wizard steps by activating the specified step.
  • populateSummaryTable(): Fills the summary table in Step 4 with data collected from previous steps.
  • saveToFile(): Exports the risk assessment data to a text file.
  • finishAssessment(): Displays a completion message and clears the page content.
